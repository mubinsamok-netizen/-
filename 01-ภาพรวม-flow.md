# 01 ภาพรวม Flow การทำงาน

## เป้าหมายใหม่

เจ้านายต้องการให้ข้อมูลจบใน note/caption ของสลิปให้มากที่สุด

ตัวอย่างการส่ง:

```text
รูปสลิป + note:
มูบิน ประชาอุทิศ 13 ค่าช่าง
```

ระบบต้องแยกให้ได้:

```text
เจ้าของอัลบั้ม = มูบิน
ไซต์ = ประชาอุทิศ 13
หมวดรายการ = ค่าช่าง
```

และอ่านจากรูป:

```text
จำนวนเงิน
วันที่เวลาโอน
ชื่อผู้โอน
ชื่อผู้รับเงิน
เลขอ้างอิง
ธนาคาร
```

## ข้อควรรู้เรื่อง LINE

แนวทางนี้ทำได้ถ้า Make เห็น note/caption มากับ image event

แต่ใน LINE Messaging API รูปแบบ image message ตามเอกสารหลักจะมีข้อมูลรูป เช่น `message.id` และ `type=image` เป็นหลัก ไม่ได้การันตีว่ามี caption อยู่ใน image event เสมอไป

ดังนั้นระบบนี้ออกแบบเป็น:

```text
แบบหลัก: image + note/caption จบใน image route
แบบสำรอง: image ก่อน แล้ว text note ตามหลัง ใช้ text route เติมแถวล่าสุด
```

## Flow หลักแบบ A-first

```text
1. เจ้านายส่งรูปสลิปใน LINE กลุ่ม พร้อม note เช่น:
   มูบิน ประชาอุทิศ 13 ค่าช่าง

2. Make รับ image event จาก LINE

3. Make ดึงรูปจาก LINE Content API

4. Make ส่งรูป + note/caption ให้ Gemini

5. Gemini อ่านข้อมูลจากรูป:
   - ยอดเงิน
   - วันที่เวลาโอน
   - ชื่อผู้โอน
   - ชื่อผู้รับเงิน
   - เลขอ้างอิง
   - ธนาคาร

6. Gemini อ่านข้อมูลจาก note/caption:
   - เจ้าของอัลบั้ม
   - ไซต์
   - หมวดรายการ

7. Make อัปโหลดรูปเข้า Google Drive

8. Make เพิ่มแถวใน Google Sheets tab รายการสลิป

9. ถ้าแยกเจ้าของอัลบั้มได้:
   - สถานะแท็ก = แท็กแล้ว
   - สถานะ = บันทึกแล้ว

10. ถ้าแยกเจ้าของอัลบั้มไม่ได้:
   - สถานะแท็ก = รอแท็กชื่อ
   - สถานะ = รอข้อมูลเพิ่ม

11. LINE bot ตอบกลับว่าสรุปอะไรได้บ้าง
```

## Flow สำรองเมื่อ LINE ไม่ส่ง note มากับรูป

ถ้า Make ไม่เห็น note/caption ใน image event ให้ใช้รูปแบบนี้:

```text
รูปสลิป
มูบิน ประชาอุทิศ 13 ค่าช่าง
```

ระบบจะทำ:

```text
1. image route บันทึกสลิปก่อน โดยสถานะแท็ก = รอแท็กชื่อ
2. text route รับข้อความ note
3. text route หาแถวล่าสุดที่รอแท็กชื่อ
4. text route เติมเจ้าของอัลบั้ม / ไซต์ / หมวดรายการ
5. เปลี่ยนสถานะแท็ก = แท็กแล้ว
```

## รองรับหลายรูปแล้ว note ครั้งเดียว

ถ้าเจ้านายส่ง:

```text
รูป 1
รูป 2
รูป 3
มูบิน ประชาอุทิศ 13 ค่าช่าง
```

ระบบควรตีความว่า:

```text
รูป 1, 2, 3 เป็นของมูบิน ไซต์ประชาอุทิศ 13 หมวดค่าช่าง
```

วิธีทำใน Make:

```text
text route:
Search Rows หาแถวที่ สถานะแท็ก = รอแท็กชื่อ
Limit เช่น 10
Update ทุกแถวที่เจอด้วย note เดียวกัน
```

## เฟสการทำงาน

### เฟส 1: รับรูป + อ่านสลิป

```text
รูปเข้า LINE
Gemini อ่านข้อมูลจากรูป
Drive เก็บไฟล์
Sheet เพิ่มแถว
LINE ตอบกลับ
```

### เฟส 2: อ่าน note/caption ใน image route

```text
ส่งรูปพร้อม note
Gemini อ่านรูป + note
Sheet ได้เจ้าของอัลบั้ม / ไซต์ / หมวด ในแถวเดียว
```

### เฟส 3: fallback text route

```text
ใช้เมื่อ LINE ไม่ส่ง note ใน image event
ข้อความ note ตามหลัง
Update แถวล่าสุดที่รอแท็กชื่อ
```

### เฟส 4: ย้ายไฟล์ตามคน

```text
เมื่อรู้รหัสคน
หาไอดีโฟลเดอร์รายคนจาก tab รายชื่อคน
หา/สร้างโฟลเดอร์ไซต์ใต้โฟลเดอร์คนนั้น
ย้ายไฟล์จาก 01_รอแท็กชื่อ ไป ชื่อคน/ไซต์
```

โครง Drive ที่ใช้:

```text
ปี/เดือน -> ชื่อคน -> ไซต์ -> รูปสลิป
```

### เฟส 5: กันซ้ำและ error handler

```text
ตรวจ message id / เลขอ้างอิง
ถ้าซ้ำไม่เพิ่มแถวใหม่
ถ้า error ให้เก็บใน tab บันทึกข้อผิดพลาด
```
